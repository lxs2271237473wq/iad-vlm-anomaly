# Stage 7.4 Multi-backbone Comparison Analysis

## 1. Purpose

This document summarizes Stage 7.4: backbone-level generalization of anomaly-crop reasoning on VisA.

The core question is whether anomaly-crop reasoning depends only on PatchCore, or whether it also works when candidate regions are generated by another anomaly localization backbone.

## 2. Backbones

| Backbone | Type | Role |
|---|---|---|
| PatchCore | Memory-bank nearest-neighbor anomaly detector | Original candidate generator |
| FastFlow | Flow-based anomaly detector | Second candidate generator for backbone-level validation |

## 5. Key Observations

1. PatchCore and FastFlow both provide near-complete anomaly candidate coverage on VisA.

2. PatchCore has slightly stronger image-level anomaly detection, while FastFlow has stronger pixel-level localization on VisA.

3. Crop-based VLM reasoning improves over full-image prompting for both backbones.

4. FastFlow candidate crops achieve the strongest binary reasoning result under inspection_binary + crop_topk_ensemble.

## 6. Paper-level Conclusion

```text
The anomaly-crop reasoning mechanism generalizes beyond PatchCore.
On VisA, both PatchCore-guided and FastFlow-guided candidate crops substantially improve CLIP-based normal/anomaly reasoning over full-image prompting.
This supports backbone-level generalization of the proposed localization-to-reasoning framework.
```
