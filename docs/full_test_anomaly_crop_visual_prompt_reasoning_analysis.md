Observations\n\n1. Full-test candidate generation fixes the previous coverage bottleneck. Candidate coverage increases to nearly all abnormal test images.\n\n2. Real anomaly crops improve defect type reasoning over full-image reasoning.\n\n3. The best Top-1 and Macro-F1 are obtained by `generic_label + crop_topk_only`.\n\n4. The strongest Top-2 result is obtained by `visual_ensemble + crop_topk_only`.\n\n5. Category-aware visual prompts are not the strongest under real PatchCore crops, even though they performed well under GT-crop upper-bound evaluation.\n\n6. This suggests that real anomaly crops are noisier than GT crops. Shorter defect-label prompts are more robust under noisy localization.\n\n## 6. Conclusion\n\nStage 6.4 is a positive result.\n\n```text\nPatchCore candidate crops with near-full test coverage improve defect type reasoning.\nThe practical classification prompt should be short and robust, while manufacturing knowledge should be reserved for explanation generation.\n```\n\n## 7. Next Direction\n\nRecommended next stage:\n\n```text\nStage 6.5: Manufacturing-aware defect explanation generation\n```\n\nThe next stage should combine:\n\n- predicted defect type from the best real-crop prompt setting\n- product category\n- PatchCore candidate region\n- manufacturing knowledge base\n\nto generate structured explanations:\n\n```text\ndefect type -> visual evidence -> possible manufacturing process -> possible cause\n```\n
